# Архив исследований — единый вход

Статический сайт на GitHub Pages с **одним экраном входа**: логотип,
под ним логин + пароль. По логину посетитель попадает в свой сценарий.

Контент всех сценариев зашифрован «конвертом» (envelope encryption) и
расшифровывается прямо в браузере (PBKDF2-SHA256 → AES-256-GCM):

- у каждого сценария свой случайный ключ контента (CEK);
- CEK заворачивается ключом каждого, кому открыт доступ (ключ = из
  «логин+пароль»).

Поэтому **мастер-логин** открывает все сценарии (витрину + каждое
исследование, переходы на месте), а **логин партнёра** — только его
исследование. На сервер публикуется только шифртекст: без логина и
пароля прочитать нельзя даже через «исходный код страницы».

## Структура

```
/index.html        единый вход + зашифрованный архив (генерируется)
/admin/index.html  админ-панель (управление пользователями)
/assets/site.css   дизайн-система (единый стиль)
/assets/bg.js      анимированный WebGL-фон (только экран входа)
/assets/anim.js    анимации (типографика, reveal, параллакс)
/assets/vault.js   вход и расшифровка по логину
/assets/admin.js   логика админ-панели
/build.mjs         сборка index.html
/_src/             ОТКРЫТЫЕ исходники + пароли — в git НЕ попадают
/robots.txt        запрет индексации
```

## Пользователи, логины, доступы

Источник правды по пользователям — зашифрованный блок `cfg` внутри
`index.html` (доступен только ролям `admin`). Управляется через
**админ-панель** (см. ниже). При первой сборке берутся `BOOTSTRAP_USERS`
из `build.mjs`, пароли — из `_src/secrets.json`:

| логин   | роль    | доступ        | куда ведёт |
|---------|---------|---------------|------------|
| admin   | admin   | hub + r1 + r2 | витрина (все исследования) |
| cyprus  | partner | r1            | «Муж на час» |
| samui   | partner | r2            | «Самуи»      |

У пользователя может быть несколько паролей и срок доступа (`exp`,
мягкая проверка на стороне клиента). Пароли хранятся в `cfg`
(зашифровано, только admin) и в `_src/secrets.json` (bootstrap, не
коммитится).

## Админ-панель `/admin/`

Открывается под мастер-логином (роль `admin`). В браузере:

- создание/удаление пользователей, нескольких паролей на пользователя
  (с генерацией), привязка к исследованиям, роль `admin`/`partner`,
  стартовая страница, **срок доступа**;
- пересобирает зашифрованный `index.html` целиком и отдаёт на скачивание.

После правок: скачать `index.html` → заменить в корне репозитория →
закоммитить. Правки переживают пересборку контента: `node build.mjs`
подхватывает текущих пользователей из `cfg` (нужен актуальный
admin-пароль в `_src/secrets.json`).

## Дизайн-система

Весь сайт оформлен в дизайн-системе **Mednov Family Office** (тёмное
«жидкое стекло»: фон `#0F2120`, бирюзовый акцент `#2CB0A8`, Montserrat +
Source Sans 3, glass-панели, pill-кнопки, фирменный знак «m.»). Токены
встроены в `assets/site.css`; логотипы — `assets/logo-mark*.svg`.

## Как редактировать / добавить исследование

```bash
node build.mjs decrypt              # восстановить _src/* (мастер-логином)
# отредактировать _src/<файл>.html
# для нового исследования: добавить файл в _src/, запись в SCENARIOS
# внутри build.mjs; карточку с data-scn — в _src/hub.html; доступы к
# нему раздать в админ-панели
node build.mjs                      # пересобрать index.html
git add -A && git commit -m "..." && git push
```

## Приватность

- Репозиторий должен быть **приватным**: Settings → General → Danger Zone →
  Change visibility → Private. Скрывает исходники и историю; Pages при этом
  продолжает публиковать только зашифрованный `index.html`.
- `noindex` + `robots.txt` запрещают индексацию.
